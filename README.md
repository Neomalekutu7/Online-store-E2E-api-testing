Overview

This project demonstrates end-to-end manual testing of an e-commerce API using the Fake Store API⁠. 

It covers the full API lifecycle: user management, product catalog, cart operations, order placement, payments, authentication, and error handling.

The goal is to showcase professional manual testing skills including functional, negative, boundary, and security testing, all documented with industry  artifacts.

Key Features Tested:

Users: Registration, login, profile retrieval

Products: List, retrieve, create, update, delete

Cart: Add, remove, view items

Orders: Place orders, retrieve order history

Authentication & Security: JWT validation, role-based access, token expiry.

Negative Scenarios: Invalid inputs, missing fields, unauthorized requests

Artifacts Included:

Requirements Traceability Matrix (RTM)
Test Plan
Test Cases (45)
Test Execution Report
Bug Report
Test Summary Report

These documents are fully structured for portfolio presentation, showing a complete testing workflow from planning to execution.

Tools Used
Postman
Fake Store API (https://fakestoreapi.com)

How Testing Was Conducted:

Testing was performed by executing all test cases against the API endpoints,

 validating:
Functional correctness
Error handling
Security and authentication
Boundary conditions and edge cases
Defects were logged in the Bug Report, and results were summarized in the Test Summary Report.

Purpose & Learning Outcomes
Demonstrates ability to perform end-to-end manual API testing
Showcases structured documentation and reporting
Highlights understanding of functional, negative, and security testing
